Collaboration Engine

목표 하나를 받으면 계획·분업·회의·종합·학습까지 자동으로 이어집니다

Agent OS AI의 핵심은 “에이전트가 여러 명 있다”가 아니라, 각 에이전트의 산출물이 다음 에이전트의 컨텍스트가 되고 마지막에 CEO가 하나의 실행안으로 합치는 구조입니다.

  • src/agents.ts는 에이전트 이름, 역할, 전문성, 페르소나를 정의
  • src/chat/corporate/specialist-loop.ts는 목표를 담당자별 작업으로 실행
  • src/chat/corporate/confer-phase.ts는 에이전트 간 짧은 회의 대화를 생성
  • src/chat/corporate/report-phase.ts는 CEO 종합 보고서로 산출물을 통합
  • src/chat/corporate/decisions-phase.ts는 결정사항을 지식폴더에 누적

Core Loop

아이디어 검증은 하나의 반복 루프입니다

Agent OS AI의 모든 폴더와 모듈은 이 루프를 빠르게 돌리기 위해 존재합니다.

Architecture

VS Code 확장 안의 실험 운영체제

Extension Host가 로컬 파일, Claude CLI, HTTP Bridge, Webview UI, 지식폴더를 묶습니다. 사용자는 채팅으로 명령하고, 에이전트는 파일을 읽고 쓰며 실험 기록을 쌓습니다.

  • 채팅 UI와 대시보드는 Webview로 제공
  • 모든 지식은 로컬 Markdown으로 저장
  • Bridge 서버는 외부 도구와 localhost로 연결
  • Git Sync는 사용자가 원할 때 지식폴더를 백업
